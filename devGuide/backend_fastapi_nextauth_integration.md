# FastAPI & NextAuth: OAuth2 Scopes Integration

When decoupling your architecture into a React frontend (NextAuth) and a FastAPI backend, the authentication flow changes. FastAPI does not generate the login screens or handle the Google/Quran.com OAuth handshakes; it simply verifies the tokens generated by the frontend.

## 1. The Authentication Flow

1.  **The Login**: The user clicks "Login with Google" in your React app. NextAuth handles the handshake.
2.  **The Token**: NextAuth generates a JWT (JSON Web Token) that contains the user's data and their "Scope" or "Role" (e.g., `{"email": "user@test.com", "role": "admin"}`). This token is signed with a secret key (`NEXTAUTH_SECRET`).
3.  **The Request**: React makes an API call to FastAPI, putting that token in the header: `Authorization: Bearer <token>`.
4.  **The Verification (FastAPI)**: FastAPI intercepts the request, reads the token using `OAuth2PasswordBearer`, verifies the signature using the same `NEXTAUTH_SECRET`, and checks if the user has the correct Scopes to access that specific route.

## 2. FastAPI Implementation (`api/auth.py`)

First, we set up the security dependencies in FastAPI. We use `SecurityScopes` to read what scopes the route requires, and we verify the JWT.

> [!NOTE]
> You will need to install PyJWT in your worker: `pip install PyJWT`

```python
from fastapi import Depends, HTTPException, status, Security
from fastapi.security import OAuth2PasswordBearer, SecurityScopes
import jwt # PyJWT
import os

# 1. Read the token from the Authorization header
# (We define the available scopes for Swagger UI documentation)
oauth2_scheme = OAuth2PasswordBearer(
    tokenUrl="token", # Not actually used for login, just for Swagger UI
    scopes={
        "user": "Standard user access (read-only)",
        "admin": "Admin access (write, approve Waiting Room)",
    }
)

# Shared secret between Next.js and FastAPI
SECRET_KEY = os.getenv("NEXTAUTH_SECRET", "super-secret-key")
ALGORITHM = "HS256"

async def get_current_user(
    security_scopes: SecurityScopes, 
    token: str = Depends(oauth2_scheme)
):
    # 1. Check if the route actually requires specific scopes
    if security_scopes.scopes:
        authenticate_value = f'Bearer scope="{security_scopes.scope_str}"'
    else:
        authenticate_value = "Bearer"

    credentials_exception = HTTPException(
        status_code=status.HTTP_401_UNAUTHORIZED,
        detail="Could not validate credentials",
        headers={"WWW-Authenticate": authenticate_value},
    )

    try:
        # 2. Decode the NextAuth JWT
        payload = jwt.decode(token, SECRET_KEY, algorithms=[ALGORITHM])
        
        email: str = payload.get("email")
        role: str = payload.get("role", "user") # Default to 'user'
        
        if email is None:
            raise credentials_exception
            
        token_scopes = [role] # NextAuth usually stores role, which maps to our scope
        
    except jwt.PyJWTError:
        raise credentials_exception

    # 3. Verify Scopes (The Magic)
    # If the route requires "admin", but the token only has "user", throw 403
    for scope in security_scopes.scopes:
        if scope not in token_scopes:
            raise HTTPException(
                status_code=status.HTTP_403_FORBIDDEN,
                detail="Not enough permissions (Invalid Scope)",
                headers={"WWW-Authenticate": authenticate_value},
            )

    return {"email": email, "role": role, "scopes": token_scopes}
```

## 3. Protecting Your Routes (`api/main.py`)

Now, you can apply these scopes to your API routes using FastAPI's `Security()` dependency.

```python
from fastapi import FastAPI, Depends, Security
from .auth import get_current_user

app = FastAPI()

# 1. PUBLIC ROUTE (No Auth Required)
@app.get("/api/search")
async def public_search(q: str):
    return {"message": "Anyone can search the database."}


# 2. PROTECTED ROUTE: User Scope Required
# Anyone logged in can access this, regardless of role.
@app.get("/api/notebooks/my-workspace")
async def get_user_notebooks(
    current_user: dict = Security(get_current_user, scopes=["user"])
):
    return {"message": f"Welcome {current_user['email']}! Here are your private notebooks."}


# 3. HIGHLY PROTECTED ROUTE: Admin Scope Required
# Only tokens containing role="admin" will pass this check.
@app.post("/api/admin/approve-category")
async def approve_category(
    category_id: str, 
    current_user: dict = Security(get_current_user, scopes=["admin"])
):
    return {"message": "Category approved by Admin."}
```

## 4. Bridging FastAPI to SurrealDB

Because FastAPI is now the secure middleman, how does it talk to SurrealDB on behalf of the user?

Since FastAPI already verified the user's identity, you don't actually need the React frontend to pass the token directly to SurrealDB anymore. FastAPI can connect to SurrealDB as a root user, but query data specifically for the logged-in user.

```python
from surrealdb import Surreal
from fastapi import Depends

async def get_db():
    async with Surreal("ws://surrealdb:8000/rpc") as db:
        await db.signin({"user": "root", "pass": "root"})
        await db.use("bayan", "knowledge_graph")
        yield db

@app.get("/api/my-fawaid")
async def get_fawaid(
    current_user: dict = Security(get_current_user, scopes=["user"]),
    db: Surreal = Depends(get_db)
):
    # FastAPI securely passes the user's email/ID into the SurrealQL query
    query = f"SELECT * FROM items WHERE type = 'faidah' AND owner_email = $email;"
    
    # SurrealDB trusts FastAPI, and FastAPI trusts the NextAuth token!
    result = await db.query(query, {"email": current_user['email']})
    
    return result
```

## Why this is the ultimate architecture:

*   **Frontend Speed**: NextAuth handles the messy OAuth redirects without bugging your backend.
*   **Backend Security**: FastAPI strictly enforces who can execute Heavy Python tasks (like exporting PDFs or approving tags) using declarative scopes.
*   **Database Simplicity**: SurrealDB doesn't have to handle complex JWT verification because FastAPI acts as an impenetrable shield in front of it.
