## 🛠 Incident Report & Resolutions

During the rapid expansion of the dashboard, several architectural and syntax incidents were identified and resolved:

### 1. The "EOF" Syntax Incident
*   **Problem**: `Build Error: Expected '</', got '<eof>'` in `page.tsx`.
*   **Cause**: Incorrectly escaped template literals in JSX `style` props and a missing closing brace in a nested component.
*   **Fix**: Refactored JSX attributes to use standard string concatenation and verified all component boundaries.

### 2. The Client-Side Connectivity Gap (Critical)
*   **Problem**: Data Browser (DataTable) modal showed 0 records or failed to fetch.
*   **Cause**: The browser cannot reach the internal database IP (`192.168.100.33`) directly. Client components attempt to fetch from the browser's network context.
*   **Fix**: Implemented a **Server-Side API Proxy** at `/api/query`. The browser now sends SQL to the Next.js server, which performs the query within the secure devserver network and returns the result.

### 3. Missing Component Dependencies
*   **Problem**: `ReferenceError: RefreshCw is not defined` and `Button is not defined`.
*   **Cause**: Rapid UI prototyping led to missing imports from `lucide-react` and `@/components/ui/button`.
*   **Fix**: Unified imports in `MonitorClient.tsx`.

### 4. Mermaid Hydration/Async Issue
*   **Problem**: Mermaid diagrams displayed `[object Promise]`.
*   **Cause**: `beautiful-mermaid`'s `renderMermaid` can return a Promise in certain environments, causing React to render the raw object.
*   **Fix**: Implemented an `async useEffect` hook in the client component to await the SVG generation before rendering.
